  <action>
    Create two data-fetching hooks following the existing usePlayerSearch pattern:

    **useClubList.ts:**
    - Export `function useClubList(): { clubs: ClubInfo[]; loading: boolean; error: string | null }`
    - Import `ClubInfo` from `@shared/types`
    - On mount (`useEffect` with empty deps), fetch `GET /api/clubs`
    - Parse response as `ClubInfo[]`
    - Track loading (starts true, false after response) and error state
    - On error, set error to `'שגיאה בטעינת רשימת המועדונים'` (Hebrew: "Error loading club list")
    - No AbortController needed (single fetch on mount, no re-fetching)
    - No debouncing needed (from RESEARCH: "~200 clubs filter instantly")

    **useClubSearch.ts:**
    - Export `function useClubSearch(clubId: number | null, maxAge: number | null): { results: ClubSearchResult[]; loading: boolean; error: string | null; search: () => void }`
    - Import `ClubSearchResult` from `@shared/types`
    - The hook does NOT auto-fetch on param changes (unlike usePlayerSearch). Instead, expose a `search()` function that triggers the fetch. This is because the user clicks a search button (D-03) rather than auto-searching on filter change.
    - When `search()` is called: if `clubId` is null, do nothing. Otherwise, build URL params: `club=${clubId}` and if maxAge is not null, add `minAge=0&maxAge=${maxAge}` (per D-06: "API call sets minAge=0 and maxAge to the entered value").
    - Fetch `GET /api/clubs/search?${params}`
    - Use AbortController (from useRef) to cancel in-flight requests when search() is called again
    - On success, parse as `ClubSearchResult[]` and set results
    - On error, set error to `'שגיאה בחיפוש'` (Hebrew: "Search error")
    - On AbortError, ignore (don't set error)
    - loading starts false, set true when search begins, false in finally block
    - Return `{ results, loading, error, search }`
  </action>
