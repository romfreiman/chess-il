    Root cause: HeroSearch validation is gate-based only (disabled button). No visible feedback about why button stays disabled.

    In client/src/components/search/HeroSearch.tsx:
    1. Add a derived boolean after the existing `isValid` line:
       `const hasNonNumeric = query.length > 0 && !/^\d+$/.test(query);`
    2. After the closing </form> tag and before the <RecentSuggestions> component, add a conditional error message:
       ```
       {hasNonNumeric && (
         <p className="text-sm text-red-500 dark:text-red-400 mt-2 text-start" role="alert">
           מספר שחקן חייב להכיל ספרות בלבד
         </p>
       )}
       ```
       The message translates to: "Player number must contain digits only".
       Use role="alert" for accessibility so screen readers announce the error.

    In client/src/__tests__/HeroSearch.test.tsx:
    3. Add a new test after the "search button is disabled when input is non-numeric" test:
       ```typescript
       it('shows validation error message when input contains non-numeric characters', async () => {
         const user = userEvent.setup();
         renderHeroSearch();
         const input = screen.getByPlaceholderText('הזינו מספר שחקן');

         await user.type(input, 'abc');
         expect(screen.getByRole('alert')).toHaveTextContent('מספר שחקן חייב להכיל ספרות בלבד');
       });
       ```
    4. Add a second new test:
       ```typescript
       it('does not show validation error when input is empty', () => {
         renderHeroSearch();
         expect(screen.queryByRole('alert')).not.toBeInTheDocument();
       });
       ```
    5. Add a third new test:
       ```typescript
       it('does not show validation error when input is valid numeric', async () => {
         const user = userEvent.setup();
         renderHeroSearch();
         const input = screen.getByPlaceholderText('הזינו מספר שחקן');

         await user.type(input, '12345');
         expect(screen.queryByRole('alert')).not.toBeInTheDocument();
       });
       ```

    Do NOT change any existing test assertions. Only add new tests.
