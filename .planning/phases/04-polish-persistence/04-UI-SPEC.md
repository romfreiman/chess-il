## Error State Specifications

### ErrorState Component

- **Layout:** Centered vertically within the `max-w-5xl mx-auto px-4 py-8` container
- **Icon:** `AlertCircle` from lucide-react, `h-12 w-12 text-gray-400 dark:text-gray-500`
- **Heading:** 20px bold (text-xl font-bold), `text-gray-700 dark:text-gray-200`, margin-top 16px
- **Body text:** 14px normal (text-sm), `text-gray-500 dark:text-gray-400`, margin-top 8px, max-width 320px
- **Retry button:** `bg-primary text-white font-bold text-sm px-4 py-2 rounded-lg hover:bg-primary/90 transition-colors`, margin-top 16px

### Error Messages (Hebrew)

| Scenario | Heading | Body |
|----------|---------|------|
| Player not found (404) | שחקן לא נמצא | מספר השחקן שהוזן לא קיים במערכת. בדקו את המספר ונסו שוב. |
| Network/server error | שגיאה בטעינת נתונים | לא הצלחנו להתחבר לשרת. בדקו את החיבור לאינטרנט ונסו שוב. |

### Error Detection

- `usePlayer` hook already returns `error` string
- PlayerPage differentiates by checking if error message contains "not found" or HTTP status from ApiError
- Retry button calls `refresh()` from `usePlayer` hook (Decision D-06)

## Toast Specifications

### Toast Component

- **Position:** Fixed, bottom-center: `fixed bottom-6 inset-x-0 flex justify-center z-50`
- **Appearance:** `bg-gray-900 dark:bg-gray-100 text-white dark:text-gray-900 text-sm font-bold px-4 py-3 rounded-lg shadow-lg`
- **Animation:** Fade in from bottom: CSS transition `opacity` + `translate-y` over 200ms ease-out. Fade out same in reverse.
- **Auto-dismiss:** 3 seconds (Decision D-10)
- **Max width:** `max-w-sm` (384px)
- **Implementation:** CSS transitions with React state, no library dependency (Claude's discretion from CONTEXT.md)
- **Dark mode:** Inverted contrast (light toast on dark background)
- **Accessibility:** `role="status"` and `aria-live="polite"` for screen reader announcement

### Toast Messages (Hebrew)

| Trigger | Message |
|---------|---------|
| Save limit reached (10 players) | ניתן לשמור עד 10 שחקנים |

## Save Button Specifications

### Placement
- Inside `PlayerHeader`, positioned at `absolute top-4 end-4` alongside the existing refresh button
- Save button sits to the start (right in RTL) of the refresh button: two buttons with `gap-2`

### States

| State | Label | Appearance |
|-------|-------|------------|
| Not saved | שמור | `text-gray-400 hover:text-primary transition-colors` with `Bookmark` icon (lucide-react), `w-5 h-5` |
| Saved | שמור | `text-primary` with `BookmarkCheck` icon (lucide-react), `w-5 h-5` |
| Limit reached + not saved | שמור | `text-gray-300 dark:text-gray-600 cursor-not-allowed` with `Bookmark` icon, triggers toast on click |

### Interaction
- Click toggles save/unsave via `useSavedPlayers` hook
- When `isFull` is true and player is not saved, clicking shows the limit toast
- `aria-label`: "שמור שחקן" (not saved) / "הסר שמירה" (saved)

## Remove Button on PlayerCard

### Placement
- Top-end corner of the card: `absolute top-3 end-3`
- Card container needs `relative` class added

### Appearance
- `X` icon from lucide-react, `w-4 h-4`
- `text-gray-400 hover:text-negative transition-colors`
- `aria-label="הסר שחקן"`

### Interaction
- Click calls `removePlayer(player.id)` from `useSavedPlayers` hook
- Must call `e.preventDefault()` and `e.stopPropagation()` to prevent Link navigation
- No confirmation dialog (remove is non-destructive -- player can be re-saved)

## Interaction States

### Loading Flow
1. User navigates to `/player/:id`
2. `usePlayer` hook sets `loading: true`
3. PlayerPage renders all 5 skeleton components in the same layout grid as the real dashboard
4. Data arrives: skeletons replaced with real components (no transition animation needed)

### Error Flow
1. `usePlayer` hook sets `error` string
2. PlayerPage renders `ErrorState` component centered in content area
3. User clicks "נסה שוב" -- calls `refresh()`, flow returns to Loading
4. If error persists, same ErrorState is shown again

### Save Flow
1. Player dashboard loaded -- save button shows `Bookmark` icon in gray
2. User clicks save -- `savePlayer({ id, name, rating, savedAt })` called
3. Button toggles to `BookmarkCheck` in primary blue
4. User navigates home -- player appears in PlayerGrid
5. If 10 players saved, clicking save on 11th shows toast, no save occurs

### Remove Flow
1. Home page shows saved player cards
2. User clicks X on a card -- `removePlayer(id)` called immediately
3. Card disappears from grid
4. If this was the last saved player, EmptyState renders
