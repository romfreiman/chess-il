## Layout Contract

### Desktop (768px+)

```
+--------------------------------------------------+
|  [PlayerPicker A]          [PlayerPicker B]       |
+------------------------+--------------------------+
|  PlayerHeader A        |  PlayerHeader B           |
+------------------------+--------------------------+
|  MetricCards A (2x2)   |  MetricCards B (2x2)      |
+------------------------+--------------------------+
|  Combined Rating Chart (full width)               |
+--------------------------------------------------+
```

- Two columns: `grid grid-cols-2 gap-6`
- Each column contains its picker, header, and metrics
- Combined chart spans full width below the grid
- Max width: `max-w-5xl mx-auto px-4 py-8`
- Section spacing: `space-y-6` (24px vertical gaps)

### Mobile (< 768px)

```
+--------------------------------------------------+
| [Tab: Player A]  [Tab: Player B]                  |
+--------------------------------------------------+
|  PlayerHeader (active tab)                        |
|  MetricCards (active tab)                         |
+--------------------------------------------------+
|  Combined Rating Chart (full width)               |
+--------------------------------------------------+
```

- Tab bar: Two equal-width buttons, active tab shows primary-colored bottom border (3px)
- Active tab button: `text-primary border-b-[3px] border-primary font-semibold`
- Inactive tab button: `text-gray-500 dark:text-gray-400`
- Tab bar container: `flex border-b border-gray-200 dark:border-gray-700`
- Each tab button: `flex-1 py-3 text-center text-base` (44px+ touch target)
- Tab content switches via React state (`activeTab: 'a' | 'b'`), not swipe gesture library (keeps dependencies minimal)
- Combined chart always visible below tabs, not affected by tab switching

## Player Picker Contract

### Behavior
- Renders as a `<select>` styled dropdown showing saved player names
- Default option: "-- בחרו שחקן --" (placeholder, disabled)
- Options populated from `useSavedPlayersContext().savedPlayers`
- On selection, triggers `usePlayer(selectedId)` to fetch data
- Player A picker excludes the player selected in Player B (and vice versa) to prevent comparing a player to themselves

### Styling
- Container: `bg-white dark:bg-gray-800 rounded-xl shadow-sm p-4`
- Select element: `w-full p-2 rounded-lg border border-gray-300 dark:border-gray-600 bg-white dark:bg-gray-700 text-gray-900 dark:text-gray-50 text-base`
- Label above select: `text-sm font-semibold text-gray-500 dark:text-gray-400 mb-2`
- Label text: "שחקן A" / "שחקן B"

## Combined Rating Chart Contract

### Two-Series AreaChart
- Uses Recharts `AreaChart` with two `Area` components
- Player A: stroke `#378ADD` (primary), gradient fill `ratingGradientA` (primary at 0.2 opacity fading to 0)
- Player B: stroke `#A855F7` (purple-500), gradient fill `ratingGradientB` (purple at 0.2 opacity fading to 0)
- Stroke width: 2px for both lines
- Dots: Player A `fill: #378ADD, r: 4`, Player B `fill: #A855F7, r: 4`
- Active dot: `r: 6` for both
- Shared Y-axis: `domain={['dataMin - 50', 'dataMax + 50']}` across both series
- X-axis: `tickFormatter={formatMonthYear}` (reuse from RatingChart)
- CartesianGrid: `strokeDasharray="3 3"` with `stroke-gray-200 dark:stroke-gray-700`
- No line/bar toggle on compare chart (line only for clarity)

### Chart Legend
- Positioned above chart, end-aligned (`flex justify-end gap-4 mb-2`)
- Each legend item: colored dot (8px circle) + player name in text-sm
- Player A: blue dot + name
- Player B: purple dot + name

### Chart Tooltip (two-player)
- Same card styling as single-player tooltip: `bg-white dark:bg-gray-800 shadow-lg rounded-lg p-3 border border-gray-200 dark:border-gray-700`
- Shows date (formatted with `formatMonthYear`) at top
- Below date, two rows:
  - Blue dot + Player A name + rating value (bold)
  - Purple dot + Player B name + rating value (bold)
- If only one player has data at that date, show only that row

### Data Merging
- Merge both players' chart data points by date
- Each data point: `{ date: string, ratingA: number | null, ratingB: number | null }`
- Use `connectNulls` on each Area to handle gaps where one player has no data point

### Chart Height
- `ResponsiveContainer width="100%" height={300}` (same as single-player chart)

### Chart Heading
- `text-xl font-bold text-gray-900 dark:text-gray-50`: "השוואת דירוג"

## State Machine

### Page States

| State | Condition | What Renders |
|-------|-----------|-------------|
| `empty` | Neither player selected | Two pickers with placeholder options, no chart |
| `one-selected` | One player selected, one empty | Selected player's header+metrics visible, other picker at placeholder, no chart |
| `both-loading` | Both players selected, both loading | Skeleton loaders in both columns, chart skeleton |
| `partial-loaded` | One loaded, one loading | Loaded player shows data, loading player shows skeleton, chart skeleton |
| `both-loaded` | Both players have data | Full view with headers, metrics, and combined chart |
| `error-one` | One player errored, other loaded/loading | ErrorState in errored column, data/skeleton in other, no chart |
| `error-both` | Both players errored | ErrorState in both columns, no chart |

### Tab State (mobile only)
- `activeTab: 'a' | 'b'` — controls which player's header+metrics are visible
- Default: `'a'`
- Chart visibility: always visible regardless of active tab

## Interaction Contract

### Player Selection Flow
1. User arrives at `/compare` page
2. Two pickers shown (desktop: side by side, mobile: above tab content)
3. User selects Player A from dropdown -- data fetches, header+metrics render
4. User selects Player B from dropdown -- data fetches, combined chart appears
5. Changing either picker re-fetches that player's data and updates chart

### Mobile Tab Interaction
- Tapping a tab switches the visible header+metrics section
- No animation/transition on tab switch (instant swap via conditional rendering)
- Tab bar stays fixed at top of content area (not sticky/fixed to viewport)

### URL State
- Route: `/compare` (no query params needed for picker state)
- Player selections are transient (not persisted in URL or localStorage)
- Navigating away and back resets selections

| Keyboard | Tab navigation through pickers and tab buttons, Enter/Space activates tabs |
