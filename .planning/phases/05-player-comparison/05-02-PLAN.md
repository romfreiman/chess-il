Helper function to render player content (shared between desktop columns and mobile tabs):
```typescript
function renderPlayerContent(
  data: ApiResponse | null,
  loading: boolean,
  error: string | null,
  refresh: () => void,
  isSelected: boolean
): JSX.Element | null {
  if (!isSelected) return null;
  if (loading) return <><PlayerHeaderSkeleton /><MetricCardsSkeleton /></>;
  if (error) {
    const errorType = error.toLowerCase().includes('not found') || error.includes('לא נמצא')
      ? 'not-found' as const : 'network' as const;
    return <ErrorState errorType={errorType} onRetry={refresh} />;
  }
  if (data) return <>
    <PlayerHeader ... /><MetricCards ... />
  </>;
  return null;
}
```

Note on MetricCards: The existing MetricCards uses `grid-cols-2 md:grid-cols-4`. When placed inside a compare two-column layout, the `md:grid-cols-4` will make metric cards too narrow. Since MetricCards is reused and we do NOT want to modify it (per UI-SPEC "no modification"), the 2x2 grid at mobile breakpoint will naturally apply within each column. At desktop the `md:grid-cols-4` will create a 4-column row within each compare column. If this looks too cramped, this is acceptable per D-03 which says "each with player header + metric cards" -- the user expects this layout. The metric cards will shrink to fit but remain readable at `max-w-5xl` (1024px), giving each column ~488px which accommodates 4 metric cards at ~110px each.

Page structure per UI-SPEC Layout Contract:
```html
<div className="max-w-5xl mx-auto px-4 py-8 space-y-6">
  {/* Page heading */}
  <h1 className="text-2xl font-bold text-gray-900 dark:text-gray-50">השוואת שחקנים</h1>

  {/* Guard: not enough saved players */}
  {savedPlayers.length < 2 ? (
    <div className="text-center py-12">
      <p className="text-gray-500 dark:text-gray-400">
        {savedPlayers.length === 0
          ? 'אין שחקנים שמורים. שמרו לפחות 2 שחקנים מדפי השחקנים כדי להשוות.'
          : 'שמרו עוד שחקן אחד כדי להשוות'}
      </p>
    </div>
  ) : (
    <>
      {/* Pickers row -- desktop: side by side, mobile: stacked */}
      <div className="grid grid-cols-1 md:grid-cols-2 gap-6">
        <PlayerPicker
          label="שחקן A"
          selectedId={selectedA}
          excludeId={selectedB}
          savedPlayers={savedPlayers}
          onChange={setSelectedA}
        />
        <PlayerPicker
          label="שחקן B"
          selectedId={selectedB}
          excludeId={selectedA}
          savedPlayers={savedPlayers}
          onChange={setSelectedB}
        />
      </div>

      {/* CompareHeader: headers + metrics in columns/tabs */}
      {(selectedA || selectedB) && (
        <CompareHeader
          playerAData={playerA.data}
          playerALoading={playerA.loading}
          playerAError={playerA.error}
          playerARefresh={playerA.refresh}
          playerBData={playerB.data}
          playerBLoading={playerB.loading}
          playerBError={playerB.error}
          playerBRefresh={playerB.refresh}
          isSaved={isSaved}
          isFull={isFull}
          onSave={(p) => savePlayer(p)}
          onUnsave={(id) => removePlayer(id)}
          selectedA={selectedA}
          selectedB={selectedB}
        />
      )}

      {/* Combined chart -- only when both selected */}
      {selectedA && selectedB && (
        playerA.loading || playerB.loading ? (
          <RatingChartSkeleton />
        ) : playerA.data && playerB.data ? (
          <CompareChart
            playerAName={playerA.data.player.name}
            playerBName={playerB.data.player.name}
            dataA={resolveChartData(playerA.data)}
            dataB={resolveChartData(playerB.data)}
          />
        ) : null
      )}
    </>
  )}
</div>
```

**Chart data resolution** (per Research Open Question 2 -- use ratingHistory if available, fallback to buildChartData):
```typescript
function resolveChartData(data: ApiResponse): RatingHistoryEntry[] {
  if (data.ratingHistory && data.ratingHistory.length > 0) {
    return data.ratingHistory;
  }
  // Fallback: compute from tournaments
  const computed = buildChartData(data.tournaments, data.player.rating);
  return computed.map(p => ({ date: p.date, rating: p.rating }));
}
```

**Save handler** (follow PlayerPage.tsx pattern):
```typescript
const handleSave = (player: { id: number; name: string; rating: number; club: string | null }) => {
  savePlayer(player);
};
```
