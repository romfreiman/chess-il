<objective>
Scaffold the project, define shared TypeScript types, and build the complete scraper as a pure function that converts chess.org.il HTML into structured typed data.

Purpose: This is the foundation of the entire data pipeline. The scraper is the most complex and fragile component -- it must correctly handle Hebrew content-based selectors, RTL rating change format (VALUE+/VALUE-), dd/MM/yyyy dates, optional fields, and non-existent player detection. Getting this right first (with tests) de-risks everything downstream.

Output: Working project scaffold with TypeScript, vitest, shared types, and a fully tested scraper that passes unit tests against real HTML fixtures.
</objective>

    **Step 2: Implement (GREEN phase)**

    Create `src/scraper/parse.ts`:
    - `export function parsePlayerPage(html: string): PlayerData`
    - Use `cheerio.load(html)` (import * as cheerio per Cheerio 1.x)
    - Content-based selectors per D-01:
      - Player name: `$('.p-profile .section-title h2').first().text().trim()`
      - Grade: `$('.p-profile > p').first().text().trim()`
      - Player info fields: find `<li>` by Hebrew label text in `div.full-block li`
      - Labels: 'מספר שחקן:', 'מספר שחקן פיד"ה:', 'מועדון:', 'שנת לידה:', 'תוקף כרטיס שחמטאי', 'מד כושר ישראלי', 'דירוג בישראל'
    - FIDE ID: extract `<a>` text from FIDE label li, return null if empty string
    - Rating: extract from span text, strip non-numeric
    - Expected rating: extract from `<a>` inside rating li, return null if NaN
    - Tournament table: find `<table>` by header content ('תחרות' and 'תאריך התחלה'), skip `.GridPager` rows
    - Rating change parser: regex `/^([\d.]+)([+-])$/` for VALUE+/VALUE- format
    - Result parser: regex `/\+(\d+)-(\d+)=(\d+)/` for +W-L=D format
    - Date parser: split on '/', reconstruct as YYYY-MM-DD (DO NOT use `new Date()`)
    - Update date: check for 'בעדכון הבא' literal -> isPending=true, otherwise extract date with regex `/(\d{2}\/\d{2}\/\d{4})/`
    - Tournament URL: prepend `https://www.chess.org.il` and replace leading `..` if present
    - Throw error if name is empty or rating is NaN (minimum viable fields per D-03)
    - Set `scrapedAt` to `new Date().toISOString()`

    Create `src/scraper/fetch.ts`:
    - `export async function fetchPlayerPage(id: number): Promise<string>`
    - Use axios GET to `https://www.chess.org.il/Players/Player.aspx?Id=${id}`
    - Headers: `{ 'User-Agent': 'ChessIL-Dashboard/1.0 (community project)' }` per D-04
    - Timeout: 15000ms
    - responseType: 'text'
    - On HTTP 500: check response for "Runtime Error" text, throw error with code PLAYER_NOT_FOUND
    - On network error: throw with descriptive message including player ID
    - Use `validateStatus: () => true` to handle 500 without axios throwing

    Create `src/scraper/validate.ts`:
    - `export function validatePlayerData(data: PlayerData): PlayerData`
    - Verify name is non-empty string
    - Verify id is positive integer
    - Verify rating is positive number
    - Verify tournaments is an array
    - Return data if valid, throw if not

    Create `src/scraper/index.ts`:
    - `export async function scrapePlayer(id: number): Promise<PlayerData>`
    - Orchestrate: fetchPlayerPage(id) -> parsePlayerPage(html) -> validatePlayerData(data)
    - Re-export fetchPlayerPage, parsePlayerPage for direct access
