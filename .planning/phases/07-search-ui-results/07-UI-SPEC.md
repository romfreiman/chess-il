# Phase 7 — UI Design Contract

> Visual and interaction contract for the Club Search UI & Results phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, checkbox-to-label gap |
| sm | 8px | Compact element spacing, input internal gaps |
| md | 16px | Default element spacing, card padding on mobile, form field gaps |
| lg | 24px | Section padding, card padding on desktop |
| xl | 32px | Gap between filter form and results table |
| 2xl | 48px | Top margin below tabs |
| 3xl | 64px | Not used this phase |

Exceptions: Floating action bar uses 24px bottom offset from viewport (matching existing compare button at `bottom-6`).

---

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 (normal) | 1.5 | `text-base font-normal` |
| Label | 14px | 400 (normal) | 1.5 | `text-sm font-normal` |
| Heading | 20px | 700 (bold) | 1.2 | `text-xl font-bold` |
| Small | 12px | 400 (normal) | 1.5 | `text-xs font-normal` |

Notes:
- Tab labels use `text-base font-bold` when active, `text-base font-normal` when inactive (matches CompareHeader tab pattern).
- Result count text uses `text-sm font-bold` for the number, `text-sm font-normal` for surrounding copy.
- Table header cells use `text-xs font-bold` uppercase-style.
- Floating action bar text uses `text-lg font-medium` (matches existing compare button).

## Interaction States

### Tab Switching
- Default active tab: Player search (`חיפוש שחקן`)
- URL param `?tab=clubs` activates club search tab
- Tab switch is instant — no transition animation
- Switching tabs preserves URL params for each tab independently
- Club search tab pre-loads club list on first activation (not on every switch)

### Search Form
- Search button is disabled until a club is selected
- Submitting updates URL to `/?tab=clubs&club={id}&maxAge={age}`
- Age field is optional — omitting it sends no age filter
- New search clears previous selection state (from Specifics)

### Combobox
- Opens on input focus
- Filters as user types (client-side filter of pre-loaded list)
- Closes on item selection, blur, or Escape key
- Shows all clubs when input is empty and focused
- Selected club name displays in input after selection

### Selection
- Checkbox click toggles individual row selection
- Select-all toggles all results; if some selected, first click selects all, second deselects all
- Selection persists within search session, resets on new search
- Floating action bar appears/disappears based on selection count (>0 / =0)

### Loading
- Skeleton replaces results area during fetch
- Filters remain visible and editable during loading
- Previous results are cleared when new search starts

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Tab bar | `role="tablist"`, `role="tab"`, `aria-selected` on each tab |
| Tab panels | `role="tabpanel"`, `aria-labelledby` linking to active tab |
| Combobox | `role="combobox"`, `aria-expanded`, `aria-autocomplete="list"`, `role="listbox"` on dropdown |
| Table | `<table>` with `<thead>` and `<tbody>`, semantic `<th>` headers |
| Checkboxes | Native `<input type="checkbox">` with `aria-label` |
| Selected rows | `aria-selected="true"` on `<tr>` elements |
| Player links | Standard `<a>` / `<Link>` elements, no onClick-only navigation |
| Floating bar | `role="status"` with `aria-live="polite"` for count updates |
| Loading | `aria-busy="true"` on results container during fetch |
| Focus management | Focus remains on search button after submit; no focus steal |
