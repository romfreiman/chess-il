## Copywriting Contract

| Element | Copy (Hebrew) | Context |
|---------|---------------|---------|
| Export button label | `ייצוא CSV` | Button text in floating bar (unchanged from Phase 7 placeholder) |
| Export button aria-label | `ייצוא שחקנים נבחרים לקובץ CSV` | Accessible label for screen readers |
| Floating bar count | `נבחרו {N} שחקנים` | Selection count text (unchanged) |
| CSV column: Name | `שם` | CSV header row |
| CSV column: Player ID | `מספר שחקן` | CSV header row (D-04) |
| CSV column: Rating | `דירוג` | CSV header row (D-04) |
| CSV column: Club | `מועדון` | CSV header row (D-04) |
| CSV column: Age | `גיל` | CSV header row (D-04) |
| CSV column: Rank | `מיקום` | CSV header row (D-04) |
| Null rating display | (empty string) | When `rating` is `null` in CSV output |
| Null age display | (empty string) | When `birthYear` is `null` in CSV output |
| Filename format | `{clubname}-{YYYY}-{MM}-{DD}-{HHmm}.csv` | e.g. `אליצור-ירושלים-2026-04-25-1430.csv` (D-09) |

No empty state, error state, or destructive action copy needed for this phase:
- **Empty state:** Not applicable -- export button only appears when 1+ players are selected (D-01). The floating bar itself is hidden at count=0.
- **Error state:** Not applicable -- client-side Blob generation cannot fail in a user-facing way. No network call involved (D-11).
- **Destructive actions:** None. Export is a read-only download operation.

## Interaction States

### Export Button States

| State | Visual | Behavior |
|-------|--------|----------|
| Default | White text on accent background, full opacity | Awaiting click |
| Hover | `bg-white/20` overlay | Cursor: pointer |
| Active/Pressed | `bg-white/30` overlay | Click triggers download |
| Focus | `ring-2 ring-white/50` outline | Keyboard accessible |

### Export Flow (User Perspective)

1. User is on club search tab with search results displayed
2. User selects 1 or more players via checkboxes (or select-all)
3. Floating bar appears showing `"נבחרו {N} שחקנים"` with enabled `"ייצוא CSV"` button
4. User clicks export button
5. Browser initiates download of CSV file immediately (no confirmation dialog, no toast -- D-10)
6. Browser's native download notification confirms the download
7. Floating bar remains visible; user can continue selecting/deselecting and export again

### Export Data Contract

| Aspect | Specification |
|--------|---------------|
| Data source | Selected players from current search results, filtered by `selected` Set (D-02) |
| Sort order | Matches current table sort order at time of export |
| Rank column | 1-based position index in the exported list (D-06) |
| Age calculation | `currentYear - birthYear` at export time (D-05) |
| Null handling | `rating: null` renders as empty string; `birthYear: null` renders age as empty string |
| Encoding | UTF-8 with BOM prefix (`﻿`) for Excel Hebrew support (D-07) |
| Escaping | RFC 4180: fields with commas, double quotes, or newlines are enclosed in double quotes; internal double quotes are doubled (D-08) |
| MIME type | `text/csv;charset=utf-8` |
| Download method | `Blob` API with `URL.createObjectURL` and programmatic `<a>` click (D-11) |
| Filename | `{clubName}-{YYYY}-{MM}-{DD}-{HHmm}.csv` using 24-hour time format (D-09) |

### "Export All" Behavior

Per D-02, there is no separate "export all" button. The user achieves "export all" by:
1. Clicking the select-all checkbox (selects all results)
2. Clicking the export button

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Export button | `<button>` element with `aria-label="ייצוא שחקנים נבחרים לקובץ CSV"` |
| Keyboard | Button is focusable via Tab, activatable via Enter/Space |
| Screen reader | aria-label provides context that visual label (`ייצוא CSV`) does not fully convey |
| Download feedback | Browser handles native download notification; no custom announcement needed |
| Floating bar | Retains `role="status"` with `aria-live="polite"` from Phase 7 |
