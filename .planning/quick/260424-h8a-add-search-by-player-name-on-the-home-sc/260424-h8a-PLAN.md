   - Uses the same axios pattern and USER_AGENT as `src/scraper/fetch.ts` (import or duplicate the constant)
   - Step 1: GET `https://www.chess.org.il/Players/SearchPlayers.aspx` to obtain ASP.NET hidden fields. Parse the HTML with Cheerio and extract the values of these hidden inputs: `__VIEWSTATE`, `__EVENTVALIDATION`, `__VIEWSTATEGENERATOR`
   - Step 2: POST to the same URL with `Content-Type: application/x-www-form-urlencoded` and body containing:
     - `__VIEWSTATE` (from step 1)
     - `__EVENTVALIDATION` (from step 1)
     - `__VIEWSTATEGENERATOR` (from step 1)
     - `ctl00$ContentPlaceHolder1$SearchNameBox` = query
     - `ctl00$ContentPlaceHolder1$SearchButton` = `חיפוש`
   - Step 3: Parse the response HTML with Cheerio. Find the results table by ID `ctl00_ContentPlaceHolder1_playersGridView`. If no table found, return empty array.
   - Step 4: For each `tr` in the table (skip the header row), extract:
     - Column 2 (index 1): Name — text content of the `a` tag. Also extract the player ID from the href `Player.aspx?Id=X` using regex `/Id=(\d+)/`.
     - Column 3 (index 2): Player number (alternative ID source, use as fallback)
     - Column 8 (index 7): Club — text content, trim, null if empty
     - Column 10 (index 9): Israeli rating — parse as integer, null if empty or NaN
     - Column 12 (index 11): Grade — text content, trim, null if empty
   - Return array of SearchResult objects. Limit to first 10 results.
   - On any error (network, parse), log the error and return empty array (do NOT throw — search should degrade gracefully).

   - Import `searchPlayers` from `../../scraper/search.js`

   - Add `playerRouter.get('/search', async (req, res) => { ... })` — this MUST be registered BEFORE the `/:id` route so Express doesn't match "search" as an ID parameter
   - Read `req.query.q` as string, trim it. If empty or length < 2, return `res.json([])` (minimum 2 chars to search).
   - Call `searchPlayers(q)` and return the results as JSON: `res.json(results)`
   - Wrap in try/catch, return empty array on error

IMPORTANT: In `src/api/routes/player.ts`, the `/search` route MUST be added BEFORE the `/:id` route. Express matches routes in order, and `/:id` would capture "search" as an ID value otherwise. Move the existing `/:id` handler below the new `/search` handler.
